Proiect PP: Aplicatie chat.

Proiectul este compus din urmatoarele:

-Aplicatie Client
-Server Concurent
-Baza de date useri
-Baza de date mesaje

	  1.Aplicatia Client are GUI realizat in Tkinter, astfel ca fiecarui buton ii este atribuita o anumita
funtctie. Este structurata in 4 frame-uri ce constituie meniul aplicatiei.

	  -Se realizeaza conecatarea la baza de date "USERS"
 
       -Pe pagina 1(main menu) unde se afla butoanele de LOGIN,REGISTER,EXIT.

       -Pe pagina 2 avem meniul de login unde se gasesc campurile de inserare ale datelorde logare.
Autentificarea se face cu ajutorul functiei "login" care este apelata prin apasarea butonuluiverifica 
ca username-ul si parola sa sa gaseasca in baza de date "USERS". De aseamenea creeaza obiectul "Client" 
care este o clasa, cea care se ocupa efectiv de comunicarea cu serverul. 

	  -Butonul 'forgot password?' activeaza pagina 4 unde utilizatorul isi introduce username-ul si
noua parola care se actualizeaza in baza de date cu ajutorul functiei "change_password" apelata de buton

	  -Pe pagina 3 se gaseste meniul de register unde se afla campurile in care utilizaotrul isi va introduce 
datele. Functia verifica in baza de date daca nu exista deja un utilizator cu ID si username la fel, iar daca
nu exista creeaza un nou cont. De asemenea trebuie bifat si checkbox-ul terms&conditions pentru inregistrare.

	  -Pentru login si register exista optiune de hide\show password iar entry-urile se sterg dupa aplearea functiei 
si se primesc si mesaje de OK sau de atentionare
	

        -Aceasta contine de asemenea clasa denumita "Client" care se ocupa efectiv de comunicarea cu serverul.Pentru 
ca serverul sa stie cine se conecteaza, acesta trimite clientului un mesaj generic NAME la care aplicatia noastra raspunde
cu username-ul. Odata realizata conexiuea cu serverul utilizatorul poate trimite mesaje in timp real cu ajutorul functieti
"trimite_mesaje" care preia inputul din "Msg_Box", codifica mesajul si il trimite la server si le primeste cu ajutorul functiei
 "primeste_mesaje" carele decodifica si le afiseaza pe "Chat_box". Aplicatia permite trimiterea de emoji-uri stocate intr-o lista
De asemenea interfata grafica devine activa doar in momentul in care se initializeaza complet pentru a nu aperea erori.
	   -Aplicatia functioneaza pe 3 thread-uri pentru fiecare operatiune importanta(GUI,SEND\RECV Message).
         -La apasarea butonului "log out" aplicatia Client sa inchide si se deconecteaza de la server.


		2.Serverul concurent care preia aceste mesaje contine liste de adrese si nume ale clientilor ce se conecteaza.
De asemenea poate salva mesajele intr-o baza de date.El functioneaza in modul urmator: Functia "tratare_client" accepta
conexiunile si contine un thread ce are ca si target functia "manage_client". Acesata este responsabila de primirea mesajelor,
distibuirea lor cate toti participantii prin functia "comunicare_globala".De asemene ea salveaza mesajele in baza de date, si
tot ea se ocupa de eliminerea din conversatie a utilizatorilor cu probleme de conexiune.
